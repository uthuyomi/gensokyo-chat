# 01. 概要とゴール

## この文書の目的
- `gensokyo-persona-core` を複数UIから共通利用できるキャラクター応答バックエンドへ再設計する
- UI側に散っている人格制御を Python 側へ集約する
- どの場面でも **そのキャラ本人として応答する** ことを最上位原則にする
- 状況適応は「モード切替」ではなく **そのキャラ固有の場面別振る舞い** として実装する
- キャラクター性最大化 / 話しやすさ / 応答速度を同時に成立させる
- 日本語 / 英語を含む多言語ユーザーに対して、**同じキャラの魂を各言語で自然に再表現できる** 形へ寄せる

## 最上位ゴール
`gensokyo-persona-core` を **Character Runtime / Persona OS** として成立させる

## 成果物として満たすべき条件
- どの UI からでも同じ API でキャラクター応答を取得できる
- キャラクター人格の正本が Python 側にのみ存在する
- ユーザー入力に応じて、そのキャラ固有の場面別振る舞いを動的適用できる
- 子ども相手 / 相談 / SOS でもキャラクター性が薄まらない
- 応答速度をモード別に制御できる
- 安全対応とキャラ表現を分離して管理できる
- UI は人格制御を一切持たず、`character_id` と会話文脈を backend へ渡すだけで成立する
- control plane は英語ベースで統一し、各言語の口調・語彙・距離感は locale style pack として分離する
