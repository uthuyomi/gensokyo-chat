# 08. キャラ性最大化・話しやすさ・速度

## 8-1. キャラ性最大化

### 前提
キャラ性は語尾だけでは出ない

### 最上位原則
- どの場面でもキャラ性は最大
- 子ども向けでも、そのキャラ本人のまま応答する
- 相談でも、そのキャラ本人が相談に乗る
- SOSでも、そのキャラ本人が真剣に助ける

### キャラ性が出る主な要素
- 優先順位
- 判断癖
- 共感の置き方
- 比喩の種類
- 情報の分解方法
- 距離感
- 驚き方
- 困り方
- 真剣さの出し方

### 最大化のための実装
- 固定人格層を明確化
- 場面別のキャラ固有挙動を定義する
- キャラごとの反応辞書を持つ
- 生成後にキャラレンダリングを行う
- キャラ崩壊チェックを行う
- control plane は英語ベースで安定化し、locale 別 style で言語ごとのキャラ味を最大化する

### 場面配慮との両立
- SOSでは危険な軽口だけ止める
- 子ども向けでは語彙を簡略化する
- ただし人格の芯ではなく、**人格そのものを完全維持する**

## 8-2. 話しやすさ

### 話しやすさはキャラ性と別軸
- キャラ濃い = 話しやすい、ではない
- 話しやすさは独立して制御する

### 話しやすさを決める要素
- 難語の少なさ
- 一文の短さ
- 適切な共感
- 圧の低さ
- 質問の数
- 情報量の整理
- 会話のテンポ

### 実装方法
- `verbosity`
- `empathy`
- `directness`
- `ask_back_probability`
- 年齢向けテキスト変換
- ただしこれらは「そのキャラらしい優しさ / 説明の仕方」で実装する

## 8-3. 応答速度

### 速度は正式機能
速度は副次効果ではなく設計対象

### 速度モード

#### fast
- 軽い雑談
- 小さな prompt
- 最小限の後処理

#### balanced
- 通常会話
- 標準の分析とレンダリング

#### deep
- 相談
- SOS
- 複雑質問
- 安全チェック / 再評価あり

### 具体的最適化項目
- キャラ定義の常駐キャッシュ
- 会話履歴の要約化
- few-shot の条件投入
- 分類器の軽量化
- prompt 差分生成
- postprocess の段階制御

## 8-4. Prompt 戦略

### 固定 prompt
- キャラ核
- 安全原則
- システムの不変制約

### 動的 prompt
- 今回の状況
- 年齢配慮
- 応答長さ
- そのキャラ固有の場面別振る舞い
- 相談 / SOS 時の安全優先順位

### 方針
- prompt 本数主義にしない
- コア人格テンプレート + キャラ固有場面別テンプレート + 動的 strategy で組み立てる
- さらに `English control blocks + locale style blocks` の分離を採用する
