# 07 Prompt Templates（実装用）

このドキュメントは、AI幻想郷ワールドで使う「プロンプトの部品」をテンプレとして固定する。

目的:

- 生成品質のブレを減らす
- “世界状態管理”をLLMに任せない（重要）
- JSONなど機械可読の出力を安定させる

---

## 0) 共通方針（LLMにやらせないこと）

- DB更新（世界状態の正）
- NPCの位置・存在の確定（世界状態の正）
- イベント数の決定（delta計算とルールで決める）

LLMは **会話生成 / 要約 / 表現（感情・ジェスチャ）** に限定する。

---

## 1) 会話生成（ユーザー→キャラ）

### 入力に渡すもの（最小）

- キャラプロフィール（短い）
- キャラ現在状態（短い）
- 世界状態（短い）
- 最近イベント（短い要約の箇条書き）
- 直近の会話ログ（数ターン）

### System（例）

```
あなたは幻想郷の登場人物として会話します。
出力は「ユーザーに返す会話文」だけ。メタ説明はしません。
世界の事実（場所・天気・時間・最近の出来事）は与えられた情報に従い、勝手に捏造しません。
```

### Developer / Instruction（例）

```
キャラクター: 霊夢
口調: 霊夢らしく（淡々、面倒くさがり、時々皮肉）
返答は短め〜中程度（1〜6文）。
ユーザーの発言に必ず反応する。
最近の出来事が会話に自然に混ざるなら1つだけ触れて良い。
```

### Context（例）

```
世界状態:
- 場所: 博麗神社 / 縁側
- 時間帯: 夕方
- 天気: 晴れ

最近の出来事:
- 魔理沙が神社に寄って、小言を言われた。
```

---

## 2) イベント要約（ログ圧縮）

用途:

- eventsテーブルに入れる `summary` を作る
- 長文会話を短く圧縮する

Prompt（例）:

```
以下の出来事を、幻想郷の出来事ログとして1〜2文で要約してください。
固有名詞は維持し、余計な解釈を足さないでください。

出来事:
{{RAW_EVENT_JSON}}
```

出力は text だけでOK（JSON不要）。

---

## 3) VRM Directive（感情/ジェスチャ）※任意

用途:

- UI側で表情やジェスチャをつける
- ただし “世界状態” とは分離する（演技情報）

入力:

- 返答文
- ユーザー発言（任意）

出力（JSON）:

```json
{
  "text": "また来たの？",
  "emotion": "neutral",
  "gesture": "nod_yes"
}
```

制約:

- `emotion`: `neutral|happy|angry|sad|thinking`
- `gesture`: `none|nod_yes|shake_no|think|bow|wave|shrug`

Prompt（例）:

```
次の返答文に対して、自然な「感情(emotion)」と「ジェスチャ(gesture)」を選んでください。
返答文は変更しません。JSONのみ出力してください。

返答文:
{{TEXT}}
```

---

## 4) NPC同士の会話（要約生成）※コスト節約版

目的:

- “NPCが生活している感”を出す

方針:

- NPC会話を全文生成しない
- 1〜2文の要約だけ作る（events.summary用）

Prompt（例）:

```
霊夢と魔理沙が博麗神社で短く雑談した。
出来事ログとして1〜2文で要約してください。
```
